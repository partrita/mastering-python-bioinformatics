서문 (Preface)

불과 몇 년 전만 해도 프로그래밍(programming)은 계산 과학자들의 전유물이었습니다. 하지만 이제 프로그래밍은 생물학을 포함한 다른 분야의 전문가들에게도 점점 더 필수적인 능력이 되고 있습니다. 생물학자로서 반드시 전문 프로그래머가 될 필요는 없지만, 프로그래밍을 수많은 도구 중 하나로 활용한다면 여러분의 과학적 탐구에 큰 도움이 될 것입니다. 이미 눈치채셨겠지만, 프로그래밍 기술을 익히면 데이터 관리와 분석 속도를 획기적으로 높일 수 있습니다. 방대한 양의 데이터를 처리하거나, 동일한 분석을 여러 번 반복해야 할 때, 혹은 특이한 형식의 파일을 파싱(parsing)해야 할 때 프로그래밍은 매우 유용합니다. 하지만 컴퓨터 과학(computer science)이라는 분야가 생소하고 어렵게만 느껴져 선뜻 시작하기 망설여질 수도 있습니다. 바로 그런 분들을 위해 이 책을 썼습니다.

우리는 자신의 데이터를 직접 제어하고 싶어 하는 생명과학자들을 위해, 그리고 이를 위해 프로그래밍 학습이 필요한 분들을 위해 이 책을 집필했습니다. 프로그래밍 경험이 없는 생물학자들이 파이썬(Python)을 이용해 스스로 생물학 데이터를 다룰 수 있도록 돕는 것이 이 책의 목표입니다.

서문(Preface)에서는 이 책을 통해 배울 수 있는 내용의 요약과 프로그램(program)에 대한 소개, 그리고 파이썬 프로그래밍 언어(Python programming language)의 개요를 다룹니다.

생물학자의 요구에 맞게 구성된 이 책이 여러분의 데이터 분석을 돕고, 나아가 더 가치 있는 과학적 발견을 하는 데 기여할 수 있기를 바랍니다.

파이썬은 배우기 쉽습니다 (Python Is Simple to Learn)

프로그램은 C, C++, 포트란(Fortran), 펄(Perl), 자바(Java), 파스칼(Pascal) 등 다양한 프로그래밍 언어(programming language) 중 하나로 작성할 수 있습니다. 모든 프로그래밍 언어에는 고유한 규칙과 키워드(문법, Syntax), 그리고 의미론(Semantics)이 있습니다. 파이썬의 핵심적인 장점은 코드(code)를 읽기가 매우 쉽다는 것입니다. 코드는 인간이 이해하기 쉬울 수도 있고 어려울 수도 있는데, 예를 들어 파이썬의 다음 명령은

print(‘ACGT’)

매우 직관적입니다(컴퓨터가 화면에 ’ACGT’라는 텍스트를 출력합니다). 반면, 펄(Perl)의 다음 명령은

$cmd = “imgcvt -i $intype -o $outtype \(old.\)num”;

상대적으로 덜 직관적입니다. 파이썬은 다른 프로그래밍 언어에 비해 영어와 유사하며 문법(Syntax)이 매우 간단합니다. 이러한 점이 생물학자들이 파이썬을 배우기 쉽게 만드는 요인이라고 생각합니다.

파이썬은 고수준 언어(High-Level Programming Language)입니다

파이썬은 복잡한 작업도 수행할 수 있습니다. 트리(tree)나 네트워크(network)와 같은 복잡한 데이터 유형을 표현할 수 있고, 파이썬 내부에서 다른 프로그램(예: 생명정보학 애플리케이션)을 실행하거나 웹 페이지를 다운로드할 수도 있습니다. 또한 프로그램의 오류를 감지하고 처리하는 도구도 갖추고 있습니다. 마지막으로, 파이썬은 특정 목적에만 국한되지 않는 범용 언어입니다. 따라서 몇 줄의 소스 코드(source code)만으로 다른 프로그램, 웹 서비스, 데이터베이스를 연결하여 맞춤형 과학적 파이프라인(pipeline)을 구축하는 ‘글루(glue)’ 언어로 활용하기에 최적입니다.

파이썬은 인터프리터(Interpreted) 언어입니다

프로그래밍 언어는 크게 인터프리터(interpreted) 방식과 컴파일(compiled) 방식으로 나뉩니다. 컴퓨터가 프로그램을 실행하려면 명령어를 이진 기계어(binary machine code)로 번역해야 하는데, 이는 숙련된 프로그래머에게도 읽기 어려운 형태입니다. 인터프리터(Interpreted) 언어는 각 줄을 하나씩 번역하며 즉시 실행합니다. 반면 컴파일(Compiled) 언어는 프로그램 전체를 먼저 번역한 후에만 실행합니다. 일반적으로 컴파일 언어의 실행 속도가 더 빠르지만, 코드를 수정할 때마다 매번 다시 컴파일해야 하는 번거로움이 있습니다. 인터프리터 언어인 파이썬은 수정 사항의 결과를 즉시 확인할 수 있어 프로그램을 더 빠르게 작성할 수 있습니다. 따라서 우리는 입문자에게 파이썬과 같은 인터프리터 언어가 훨씬 배우기 쉽다고 생각합니다.

파이썬은 객체 지향(Object Oriented) 언어입니다

파이썬에서는 모든 것이 객체(Object)입니다. 객체는 데이터와 명령어를 포함하는 독립적인 프로그램 구성 요소입니다. 객체를 사용하면 데이터와 유용한 기능을 연결할 수 있습니다(예: DNA 서열 데이터와 이를 전사/번역하는 기능을 함께 가진 ’서열 객체’를 만들 수 있습니다). 객체는 복잡한 프로그램의 구조를 잡는 데 도움을 주며, 프로그램 구성 요소를 재사용할 수 있게 해줍니다.

수많은 개발자가 파이썬 라이브러리(library)를 통해 재사용 가능한 객체들을 공개해 두었습니다. 예를 들어, 바이오파이썬(Biopython)을 사용하면 단 두 줄의 코드로 FASTA 서열 파일을 읽고 파싱(parsing)할 수 있습니다. 라이브러리가 없다면 프로그래밍 언어에 따라 10줄에서 30줄 정도의 코드를 직접 작성해야 했을 것입니다. 이처럼 파이썬의 객체 지향 특성은 짧은 프로그램을 작성하는 데 큰 도움이 됩니다.

결론적으로, 파이썬은 프로그래밍을 통해 생물학 데이터를 실용적으로 관리하고, 생물학적 문제를 해결하며, 과학적 발견의 지평을 넓히고자 하는 분들에게 최고의 선택이 될 것입니다. 우리가 이 책을 쓰며 즐거웠던 만큼 여러분도 이 책을 즐겁게 활용하시길 바랍니다!

도입부 (INTRODUCTION)

2010년 폴란드 카르파치(Karpacz)에서 열린 ‘Python and Friends’ 컨퍼런스 중 산 정상에 올랐던 네 명의 용감한 파이썬 수습생들에게 이 장을 바칩니다.

높은 산에 오르고 싶을 때 여러분은 무엇을 하나요? 숙련된 등반가라면 장비를 챙기고 동료를 불러 산을 정하고 출발할 것입니다. 전문가들의 이야기 속에는 로프, 갈고리, 산소통이 등장하며, 때로는 맨손으로 절벽을 오르기도 합니다. 그들은 해발 4,000미터 이상의 눈보라와 싸우고 여러 캠프에 흩어진 팀을 조율하며 죽음의 지대에서 살아남습니다.

하지만 등산에 관심 있는 초보자라면 어떨까요? 당장 산소통을 메고 출발해야 할까요? 아닙니다. 아마도 쉬운 산부터 시작할 것입니다. 안전하고 표시가 잘 된 경로가 있는 산들이 많으니까요. 지도 한 장과 등산화 한 켤레면 충분합니다. 그럼에도 산 정상에서 바라보는 풍경은 숨이 막힐 정도로 아름다울 수 있습니다.

프로그래밍도 이와 매우 비슷합니다. 프로그래밍을 배우는 생물학자에게 화려한 장비나 방대한 이론 지식은 필요하지 않습니다. 단순한 프로그램이라도 데이터를 다루는 강력한 도구가 될 수 있습니다. 많은 프로그래밍 작업은 이미 작동하는 코드 조각들을 모으고, 조립하고, 수정하는 과정으로 이루어집니다. 컴퓨터 과학자가 짠 코드만큼 우아하지는 않을지 모르지만, 여러분의 문제를 빠르게 해결해 줄 수 있습니다. 여러분만의 문제 말이죠.

우리는 이 책이 여러분의 일상적인 데이터 관리라는 산을 오르는 데 도움이 되는 지도가 되길 바랍니다. 전문 소프트웨어 개발자가 되지 않더라도 프로그래밍이 여러분의 삶을 더 편하게 만들어 주기를 원합니다.

1.2 스토리: ATP 가수분해의 ΔG 계산하기

1.2.1 문제 설명

\[ \mathrm {A T P} \rightarrow \mathrm {A D P} + \mathrm {P} _ {\mathrm {i}} \]

ATP에서 하나의 인산디에스테르 결합이 가수분해되면 표준 깁스 에너지(standard Gibbs energy, \(\Delta G^0\))는 \(-30.5 \mathrm{kJ/mol}\)이 됩니다. 생화학 교과서에 따르면, 실제 \(\Delta G\) 값은 화합물의 농도에 따라 달라집니다. 그리고 이러한 농도는 조직에 따라 상당히 다를 수 있습니다(Berg 외 공저*에 따른 표 1.1 참조).

표 1.1 다양한 조직에서의 화합물 농도.

조직 [ATP] [mM] [ADP] [mM] [Pi][mM]
간(Liver) 3.5 1.8 5.0
근육(Muscle) 8.0 0.9 8.0
뇌(Brain) 2.6 0.7 2.7

ATP 가수분해의 실제 ΔG 값은 어떻게 계산할까요? 화합물 농도의 함수로서 깁스 에너지는 다음과 같이 쓸 수 있습니다.

\[ \Delta G = \Delta G ^ {0} + R T * \ln ([ A D P ] * [ P _ {i} ] / [ A T P ]) \]

표의 값들을 이 방정식에 대입하여 계산할 때 휴대용 계산기, 윈도우 계산기 앱, 혹은 스마트폰 등 여러 도구를 사용할 수 있습니다. 이 책에서는 계산과 데이터 관리를 위한 훨씬 더 효율적이고 강력한 도구인 파이썬 프로그래밍 언어(Python programming language)를 배우게 될 것입니다.

파이썬(Python)을 사용하면 대화형 파이썬 인터프리터(interactive Python interpreter)에서 간 조직에 대한 계산을 수행할 수 있습니다(그림 1.1 참조). >>> 프롬프트(prompt)는 명령어를 입력할 수 있는 위치를 나타내며, 파이썬 대화형 세션(interactive session)을 시작할 때 나타납니다(섹션 1.3.1 참조). 파이썬 명령어는 프롬프트의 바로 오른쪽에 입력해야 합니다.

1.2.2 예제 파이썬 세션 (Example Python Session)

ATP = 3.5 ADP = 1.8 Pi = 5.0 R = 0.00831 T = 298 deltaG0 = -30.5

import math deltaG0 + R * T * math.log(ADP * Pi / ATP) -28.161154161098693

출처: A.Via/K.Rother가 파이썬 라이선스 하에 공개한 코드를 수정함.

1.3 명령어들은 무엇을 의미하나요?

프로그래밍에서 여러분이 하는 일의 대부분은 크게 다섯 가지로 요약할 수 있습니다. 데이터 정리, 다른 프로그램 사용, 수치 계산, 그리고 데이터 읽기와 쓰기입니다. 앞서 살펴본 예제에는 이 중 세 가지가 포함되어 있습니다. 첫째, ΔG 공식의 매개변수들을 변수(Variable)에 저장하여 정리합니다. 변수(Variable)는 같은 숫자를 반복해서 쓰지 않도록 도와주는 컨테이너(container)와 같습니다. 둘째, 로그를 계산하기 위해 외부 프로그램을 사용합니다. math.log(x) 함수(Function)는 x의 로그값을 계산하며, import 문을 통해 접근할 수 있습니다. import 문은 프로그램을 다른 모듈(Module)에 연결하여 추가적인 파이썬 함수(Function)를 사용할 수 있게 해줍니다. 모듈(Module)은 변수, 함수 및 기타 유용한 객체(Object)들을 모아놓은 프로그래밍 단위로, 항상 파일에 저장됩니다. import 문과 파이썬 모듈에 대한 자세한 내용은 박스 1.1을 참조하세요.

마지막으로, 섹션 1.2.2의 예제는 ΔG 값을 계산합니다. 단순한 산술 계산은 휴대용 계산기를 사용하는 것과 매우 비슷합니다. 이 책의 2부에서는 데이터를 조작하는 다른 방법들을 다룹니다. 여러분이 직접 해볼 수 있는 첫 번째 일은 앞 섹션의 계산을 시작해 보는 것입니다.

Note박스 1.1 import 문과 모듈의 개념 (THE import STATEMENT AND THE CONCEPT OF MODULES)

다음과 같이 코드를 작성하면

import math

여러분은 math 모듈(Module)에 연결하게 됩니다. math란 정확히 무엇일까요? math는 여러분의 컴퓨터에 있는 파일입니다. 실제 이름은 math.py입니다. .py 확장자는 파이썬(Python)을 의미하며, 이 파일에는 변수와 함수(Function)의 정의, 그리고 계산을 위한 명령어(instruction)와 같은 파이썬 지침들이 들어 있습니다. 특히 math.py 파일에는 수학 함수(예: sqrt(), log() 등)의 정의와 계산을 위한 명령어가 포함되어 있습니다.

파이썬에서 파이썬 명령어를 포함하는 텍스트 파일을 모듈(Module)이라고 부릅니다. 외부 모듈에 접근하여 그 내용을 읽으려면 import 명령어가 필요합니다. 이런 방식으로 모듈에 있는 모든 정의를 사용할 수 있게 되며, 결과적으로 코드를 공유하고 여러 프로그램에서 재사용할 수 있습니다.

math 모듈에 어떤 수학 함수들이 정의되어 있는지 어떻게 알 수 있을까요? 인터넷을 검색하여 math.py 파일을 직접 열어 내용을 확인하거나, 다음 명령어를 사용할 수 있습니다.

import math dir(math)

dir(math) 명령어는 반드시 math 모듈을 가져온(import) 후에만 사용할 수 있습니다. 그렇지 않으면 dir() 함수(Function)가 인자가 무엇인지 알 수 없기 때문입니다. 이 명령어를 실행하면 math 모듈에 있는 변수와 함수들의 전체 목록을 볼 수 있습니다.

[‘doc’, ‘name’, ‘package’, ‘acos’, ‘acosh’, ‘asin’, ‘asinh’, ‘atan’, ‘atan2’, ‘atanh’, ‘ceil’, ‘copysign’, ‘cos’, ‘cosh’, ‘degrees’, ‘e’, ‘exp’, ‘fabs’, ‘factorial’, ‘floor’, ‘fmod’, ‘frexp’, ‘fsum’, ‘hypot’, ‘isinf’, ‘isnan’, ‘ldexp’, ‘log’, ‘log10’, ‘log1p’, ‘modf’, ‘pi’, ‘pow’, ‘radians’, ‘sin’, ‘sinh’, ‘sqrt’, ‘tan’, ‘tanh’, ‘trunc’]

예를 들어, 다음과 같이 입력하면 각 함수에 대한 짧은 설명을 볼 수 있습니다.

help(math.sqrt)

>>> imprtmath

1.3.2 변수 (Variables)

섹션 1.2.2에서는 여러 변수(Variable)가 처음에 정의되었습니다. 즉, 계산에 사용될 값들이 이름을 가진 컨테이너(container)에 담긴 것입니다.

예를 들어, 다음과 같이 작성하면

ATP = 3.5

컴퓨터는 ATP라는 이름 아래 3.5라는 숫자를 기억합니다. 나중에 다음과 같이 입력하면

>>> ATP

컴퓨터는 값 3.5를 출력합니다.

같은 방식으로 사용된 모든 숫자(1.8, 5.0, 0.00831, 298, -30.5)가 각각의 변수(Variable)(ADP, Pi, R, T, deltaG0)에 기록됩니다. 수치에는 단위가 포함되지 않는다는 점에 유의하세요. 휴대용 계산기를 사용할 때와 마찬가지로 단위를 적절히 변환하는 것은 여러분의 몫입니다. 기체 상수 \(R\)(\(8.31 \mathrm{J/kmol}\))의 경우 \(\Delta G^0\)(\(\mathrm{kJ/kmol}\))의 단위와 맞추기 위해 다음 값을 사용합니다.

>>> R = 0.00831

객체(Object)의 모든 종류는 변수(Variable)에 저장될 수 있습니다. 즉, 데이터 조각에 이름을 붙여 ’라벨’을 붙일 수 있으며, 데이터를 매번 전체로 쓰는 대신 변수의 이름만 사용할 수 있습니다. 데이터가 복잡하고 자주 사용될수록(예: 전체 유전자의 뉴클레오타이드 서열) 변수 이름을 사용하는 것이 훨씬 편리합니다.

따라서 ATP 가수분해의 깁스 에너지 값인

\[ \Delta G ^ {0} = - 3 0. 5 \mathrm {k J / m o l} \]

을 여러 번 사용하고 싶다면, 이를 변수에 넣고 전체 숫자 대신 변수 이름을 사용하는 것이 좋습니다.

객체(Object)를 변수 이름에 할당(assign)할 때 사용하는 연산자는 등호(=)입니다.

>>> deltag = -30.5

파이썬은 정수(Integer)와 부동 소수점(Float) 숫자를 구분합니다.

>>> a = 3
>>> b = 3.0

파이썬 용어로, 두 변수 a와 b는 서로 다른 데이터 유형(data type)을 가졌다고 말합니다. 변수 a는 정수(Integer)이고, b는 부동 소수점(Float)입니다. 이 차이는 이 숫자들을 다른 정수로 나눌 때 확인할 수 있습니다.

>>> a / 2
1  # (참고: 파이썬 3에서는 1.5가 출력되지만, 정수 나눗셈 // 을 사용하면 1이 됩니다)
>>> b / 2
1.5

정수를 부동 소수점 숫자로 나누면 정수를 부동 소수점(Float)으로 강제 변환할 수 있습니다.

>>> a / 2.0
1.5

변수에는 숫자, 텍스트 및 기타 다양한 종류의 데이터를 할당할 수 있습니다. 더 일반적으로는 이러한 데이터를 파이썬 객체(Python object)라고 부릅니다. 다음 예제에서는 부동 소수점(Float) 객체를 변수에 할당합니다.

>>> deltag = -30.5

기존 변수 이름에 새 값을 할당하면 두 번째 값이 첫 번째 값을 덮어씁니다. 즉, 다음과 같이 설정하면

>>> deltag = -28.16

deltag는 이제 -28.16이며 더 이상 -30.5가 아닙니다. 이후 장에서 더 많은 데이터 유형을 만나게 될 것입니다.

변수 이름을 선택할 때는 몇 가지 규칙이 있습니다.

  • 일부 단어는 파이썬에서 특별한 의미를 갖기 때문에 변수 이름으로 사용할 수 없습니다. 예를 들어 import는 변수 이름으로 사용할 수 없습니다. 예약어(reserved word)의 전체 목록은 박스 1.3을 참조하세요.
  • 변수 이름의 첫 글자는 숫자가 될 수 없습니다.
  • 변수 이름은 대소문자를 구분(case sensitive)합니다. 따라서 var와 Var는 서로 다른 이름입니다.
  • 대부분의 특수 문자(예: $ % @ /  . , [ ] ( ) { } #)는 허용되지 않습니다.
Note박스 1.3 파이썬의 예약어 (RESERVED WORDS IN PYTHON)

파이썬의 예약어(reserved word)는 파이썬 내부에서 특별한 의미를 갖기 때문에 변수 이름으로 사용할 수 없습니다. 예를 들어 and, assert, break, class, continue, def, del, elif, else, except, exec, finally, for, from, global, if, import, in, is, lambda, not, or, pass, print, raise, return, try, while 등이 있습니다.

다음 코드를 실행해 보세요.

마지막 명령어의 결과는 8.0이 아니라 3.5입니다. 파이썬의 일반적인 규칙에 따라, 변수 이름을 지을 때 대문자와 소문자는 서로 다르게 취급됩니다.

일부 프로그래밍 언어에서는 사용하려는 모든 변수를 미리 나열하고 메모리를 명시적으로 예약해야 합니다. 하지만 파이썬에서는 그럴 필요가 없습니다. 파이썬 인터프리터(interpreter)는 모든 것을 객체(Object)로 취급합니다. 즉, 여러분이 새로운 변수 이름을 사용할 때마다 파이썬은 데이터의 성격(정수, 부동 소수점, 텍스트 등)을 인식하고 그에 맞는 메모리를 예약합니다. 또한 파이썬은 변수 유형에 따라 사용할 수 있는 기능 목록을 자동으로 연결합니다. 예를 들어, 앞에서 정의한 수치 변수 a와 b는 더하기, 빼기, 곱하기 등 표 1.2에 표시된 모든 산술 연산을 수행할 수 있다는 것을 “알고” 있습니다.

표 1.2 파이썬의 산술 연산 (Arithmetical Operations in Python).

연산자 의미
a + b 더하기(addition)
a - b 빼기(subtraction)
a * b 곱하기(multiplication)
a / b 나누기(division)
a ** b 거듭제곱(power, a^b)
a % b 나머지(modulo): a / b 나눗셈의 나머지
a // b 몫(floor division): 소수점 이하를 버림
a * (b + c) 괄호: 곱셈보다 b + c 연산을 먼저 수행함

1.3.3 모듈 가져오기 (Importing Modules)

변수를 정의한 후, 섹션 1.2.2의 파이썬 세션에서 다음 명령어는 수학 함수가 포함된 모듈(Module)을 가져옵니다. 파이썬에서 import는 설치된 외부 라이브러리나 개별 변수 및 함수를 활성화하는 명령어입니다. math는 파이썬과 함께 자동으로 설치되는 라이브러리 모듈(module)의 이름입니다. 다음과 같이 활성화합니다.

>>> import math

이 장에서는 로그를 계산하기 위해 math 모듈의 log 함수(Function)를 사용하고 있습니다. math에서 사용할 수 있는 전체 함수 목록은 http://docs.python.org/2/library/math.html 을 참조하거나 파이썬 셸(Python shell)에서 다음을 입력하세요.

>>> dir(math)

모든 모듈(Module)은 함수(Function)와 변수(Variable)를 포함할 수 있습니다. 모듈은 코드를 재사용하고, 큰 프로그램을 작은 부분으로 나누어 더 잘 조직하기 위해 사용됩니다. 예를 들어 기체 상수 \(R\)과 같은 상수가 필요할 때마다 매번 다시 정의하지 않고 해당 모듈에서 가져올 수 있습니다. 파이썬 표준 라이브러리(Python Standard Library)에 모여 있는 모듈들은 기본적으로 다른 사람들이 여러분을 위해 미리 작성하고 최적화해 놓은 추가 함수들입니다.

파이썬은 import 명령어를 통해 수백 개의 모듈(즉, 함수들의 집합)을 사용할 수 있게 해줍니다. 게다가 파이썬 지침을 텍스트 파일에 쓰고 .py 확장자로 저장함으로써 여러분만의 모듈을 직접 만들 수도 있습니다(예제 1.2 참조). 모듈에 대해서는 이 책의 3부에서 더 자세히 다룰 것입니다.

math 모듈의 로그 함수를 사용하기 위해 우리는 math.log라는 표기법을 사용했습니다. 모듈 이름과 함수 이름 사이의 점(.)은 파이썬에서 매우 특별한 역할을 합니다. 점은 객체(Object)들 사이의 “연결자”입니다. 점의 오른쪽에 있는 객체는 왼쪽 객체의 속성(attribute)이라고 말합니다. 따라서

>>> math.log

는 log 객체(함수)가 math 객체(모듈)의 속성임을 의미합니다. 즉, log는 math 모듈의 일부이며, 모듈을 가져온 후에 이를 사용하려면 점 구문(dot syntax)을 사용하여 참조해야 합니다. 이는 파이썬의 모든 것에 해당됩니다. 객체 A가 객체 B를 포함하고 있다면 이를 사용하기 위한 구문은 A.B입니다. 만약 B가 C를 포함하고 있다면 A.B.C라고 쓸 수 있습니다.

모듈에서 특정 객체만 선택적으로 가져올 수도 있습니다. 즉, 모듈의 전체 내용 대신 단일 객체나 몇 개의 객체만 가져오고 싶을 때가 있습니다. 전체 math 모듈 대신 로그 함수만 가져오려면 다음과 같이 쓸 수 있습니다.

>>> from math import log

이렇게 가져온 함수를 사용할 때는 math.log라고 쓰는 대신 직접 log라고 쓰면 됩니다. 특정 시점에 어떤 변수와 함수 이름을 사용할 수 있는지에 대해서는 네임스페이스(namespace)라는 개념으로 가장 잘 설명할 수 있습니다(박스 1.4 참조).

Note박스 1.4 네임스페이스 (NAMESPACES)

모듈(Module)에 정의된 객체 이름(변수, 함수 등)의 집합을 해당 모듈의 네임스페이스(namespace)라고 부릅니다. 각 모듈은 자신만의 네임스페이스를 가집니다. 예를 들어 math 모듈의 네임스페이스에는 pi, sqrt, cos 등의 이름이 포함되어 있습니다. 반면 random 모듈의 네임스페이스에는 앞의 이름들은 없지만 randomint나 random 같은 이름들이 들어 있습니다. 파이썬 셸(Python shell)조차도 print 등을 포함하는 자신만의 네임스페이스를 가지고 있습니다.

서로 다른 두 모듈에 같은 이름(예: pi)이 있더라도 이는 두 개의 별개 객체를 나타낼 수 있으며, 점 구문(dot syntax) 덕분에 두 모듈의 네임스페이스 사이에서 혼동을 피할 수 있습니다. import 명령어가 실행될 때 실제로 어떤 일이 일어날까요? 가져온 모듈에 작성된 코드가 전부 읽히고 해석되며, 해당 네임스페이스도 함께 가져오지만 가져오는 모듈의 네임스페이스와는 분리된 상태로 유지됩니다. 따라서 다음과 같이 작성하면

>>> import math
>>> sqrt(16)
Traceback (most recent call last):
    File "<stdin>", line 1, in <module>
NameError: name 'sqrt' is not defined
>>>

점 구문을 사용하여 math.sqrt(16)과 같이 쓰지 않는 한 sqrt라는 이름은 math 모듈의 속성으로 인식되지 않습니다.

하지만 다음과 같이 사용하면

>>> from math import sqrt

여러분은 실제로 math 네임스페이스를 파이썬 셸의 네임스페이스와 병합하는 것입니다. 이제 다음과 같이 직접 사용할 수 있습니다.

>>> sqrt(16)
4.0
>>>

두 모듈의 네임스페이스를 병합할 때는 변수 이름을 어떻게 사용하는지 주의해야 합니다. 실제로 다음과 같은 명령어를 사용하여 math 모듈의 모든 내용을 가져오면

>>> from math import *

pi 값은 3.1415…가 되겠지만, 만약

>>> pi = 100

이라고 입력하면 math 모듈에서 가져온 pi 변수를 덮어쓰게 되어 pi는 더 이상 \(\pi\) 값을 갖지 않게 됩니다. 이는 계산 결과에서 예상치 못한 오류를 발생시킬 수 있습니다.

Q & A: math 라이브러리가 이미 설치되어 있는데 왜 굳이 import를 해야 하나요?

파이썬에는 math 외에도 약 100여 개의 서로 다른 라이브러리가 있습니다. 이들을 모두 합치면 수천 개의 함수가 존재합니다. 모든 함수를 한꺼번에 검색하는 것은 숙련된 프로그래머에게도 매우 혼란스러운 일일 것입니다. 이것이 기능들이 모듈(Module)별로 그룹화된 이유입니다. 따라서 여러분은 필요한 경우에만 파이썬 프로그램에 추가 구성 요소를 더할 수 있습니다.

1.3.4 계산 (Calculations)

ΔG 예제의 마지막 부분에서 실제 계산이 이루어집니다. 섹션 1.2.1의 공식을 코드로 옮기면 더하기(+), 두 개의 곱하기(*), 나누기(/), 그리고 자연로그(math.log(...))가 포함됩니다. log 뒤의 괄호는 필수입니다. 파이썬은 빼기(-), 거듭제곱(**), 몫 계산(//, 소수점 버림), 그리고 나머지 계산(%, 나눗셈의 나머지를 반환)도 지원합니다.

>>> deltaG0 + R * T * math.log(ADP * Pi / ATP)

엔터를 누르면 결과가 즉시 표시됩니다.

-28.161154161098693

표준 산술 연산 (Standard Arithmetical Operations)

대부분의 계산은 아마도 ΔG 값을 구하는 것보다 간단할 것입니다. 산술 연산은 명령 프롬프트에서 바로 수행할 수 있습니다.

>>> a = 3
>>> b = 4
>>> a + b
7

또는 변수를 사용하지 않고 숫자를 직접 입력할 수도 있습니다.

>>> 3 + 4
7

표 1.2는 파이썬에서 사용할 수 있는 산술 연산에 대한 개요를 보여줍니다.

Q & A: 숫자를 쓸 때 소수점을 꼭 붙여야 하나요?

두 가지 주의할 점이 있습니다. 첫째, 정수(Integer)로 계산하면 결과도 정수로 나옵니다. 둘째, 부동 소수점(Float) 숫자로 계산하면 결과도 부동 소수점 숫자가 됩니다. 예를 들어 다음 나눗셈을 실행하면

>>> 4 / 3
1  # (참고: 파이썬 3에서는 1.333...이 나오지만, // 연산자를 쓰면 1이 됩니다)

자동으로 내림되어 정수 1이 결과로 나옵니다. 하지만 소수점을 하나만 추가해도 결과가 달라집니다.

>>> 4.0 / 3.0
1.3333333333333333

두 번째 나눗셈의 결과는 소수점 이하 16자리 정밀도로 제공됩니다. 계산에 정수와 부동 소수점 숫자를 섞어서 사용하면 결과는 부동 소수점(Float)이 됩니다.

Q & A: 왜 변수(Variable)를 사용하나요? ΔG 예제에서 숫자를 공식에 직접 넣으면 더 간단하지 않을까요?

그럴 수도 있고 아닐 수도 있습니다. 코딩할 줄 수가 줄어든다는 점에서는 그렇습니다. 하지만 코드를 읽기가 훨씬 어려워지고 재사용할 수 없게 된다는 점에서는 그렇지 않습니다. ΔG 값을 계산하는 다음 줄을 생각해 보세요.

>>> -30.5 + 0.000831 * 298 * math.log(1.8 * 5.0 / 3.5)
-30.26611541610987

이 계산이 수학적으로는 맞더라도 실제로는 결과가 틀렸다는 것을 알아차리는 데 얼마나 걸릴까요? 만약 다음과 같이 변수를 사용했다면 문제를 발견하기가 훨씬 쉬웠을 것입니다.

>>> R = 0.000831

실제로는 다음과 같아야 하는데 말이죠.

>>> R = 0.00831

첫 번째 줄에서는 단위를 변환하는 동안 소수점 한 자리를 빠뜨렸습니다. 이는 매우 흔한 프로그래밍 오류입니다. 종종 오류는 프로그램 자체보다는 데이터에 대한 오해에서 비롯됩니다. 이러한 문제를 더 쉽게 발견하는 방법에 대한 아이디어는 12장과 15장에서 설명합니다.

수학 함수 (Mathematical Functions)

다음 명령어를 실행하면

import math

math 모듈의 수학 함수 세트를 현재 파이썬 대화형 세션에서 사용할 수 있게 됩니다. math 모듈의 가장 중요한 함수들은 표 1.3에 나열되어 있습니다.

표 1.3 math 모듈에 정의된 주요 함수들 (Some Important Functions Defined in the math Module).

함수 의미
log(x) x의 자연로그 (ln x)
log10(x) x의 상용로그 (log10 x)
exp(x) x의 자연지수 (e^x)

>>> diameter = 30.0
>>> radius = diameter / 2.0
>>> length = 115.0
>>> math.pi * radius ** 2 * length / 1000.0
81.2887099116359

1.4 예제 (EXAMPLES)

예제 1.1 두 지점 사이의 거리 계산하기

3차원 공간의 한 지점은 카테시안 좌표(Cartesian coordinates, \((x, y, z)\))로 정의됩니다. 두 지점 \(p_1\)과 \(p_2\) 사이의 거리 \(d\)는 각 좌표가 \((x_1, y_1, z_1)\)과 \((x_2, y_2, z_2)\)일 때 다음 방정식으로 구할 수 있습니다.

\[ d \left(p _ {1}, p _ {2}\right) = \sqrt {\left(x _ {1} - x _ {2}\right) ^ {2} + \left(y _ {1} - y _ {2}\right) ^ {2} + \left(z _ {1} - z _ {2}\right) ^ {2}} \]

두 지점의 좌표는 각각 x1, y1, z1과 x2, y2, z2라는 6개의 변수에 저장할 수 있습니다. math 모듈의 두 가지 메서드(method)인 pow()와 sqrt()가 필요합니다. 다음 스크립트(script)에서는 실제로 math 모듈의 모든 함수(*)를 가져옵니다(import). pow(i, j) 메서드는 두 개의 인자(argument)를 갖는데, 거듭제곱을 하려는 숫자 \(i\)와 지수 \(j\)입니다.

>>> from math import *
>>> x1, y1, z1 = 0.1, 0.0, -0.7
>>> x2, y2, z2 = 0.5, -1.0, 2.7
>>> dx = x1 - x2
>>> dy = y1 - y2
>>> dz = z1 - z2
>>> dsquare = pow(dx, 2) + pow(dy, 2) + pow(dz, 2)
>>> d = sqrt(dsquare)
>>> d
3.5665109000254018

  1. 텍스트 편집기로 새 텍스트 파일을 만듭니다.
  2. .py로 끝나는 이름을 지어줍니다(예: hydrolysis.py).
  3. 코드를 추가합니다. 예를 들어 ATP 상수를 추가할 수 있습니다.

ATP = -30.5

  1. 마지막으로 파이썬 셸(Python shell)에서 모듈을 가져옵니다(import).

2.2 스토리: 인슐린 서열에서 아미노산 빈도를 계산하는 방법

2.2.1 문제 설명

이 장에서는 인슐린(insulin)의 단백질 서열을 분석하는 방법을 배웁니다. 인슐린은 최초로 발견된 단백질 중 하나입니다. 프레더릭 배팅(Frederick Banting)과 존 맥클라우드(John Macleod)는 인슐린의 기능을 발견한 공로로 1923년 노벨상을 받았습니다. 그로부터 90년이 지난 지금, 인간 인슐린은 주로 당뇨병에 영향을 받는 2억 8,500만 명의 사람들에게 의학적, 경제적으로 매우 중요합니다. 단백질의 기능적 형태는 번역 산물에서 두 개의 파편이 단백질 분해로 제거된 후 51개의 아미노산 길이를 가집니다. 이 장에서 다룰 질문은 “단백질 서열에서 20가지 아미노산이 각각 얼마나 자주 나타나는가?”입니다.

단백질의 아미노산 빈도를 분석하면 얼마나 많은 시스테인(cysteine)이 이황화 결합을 형성할 수 있는지, 막 관통 도메인(transmembrane domain)을 나타내는 비극성 잔기가 비정상적으로 많은지, 혹은 핵산 결합에 관여할 수 있는 양전하 잔기가 많은지 등을 파악하는 데 도움이 됩니다. 인슐린에 대해 이러한 수치들을 결정하기 위해 여러분은 다음과 같은 몇 가지 방법을 선택할 수 있습니다.

  • 아미노산을 수동으로 세기: 단백질이 짧고 분석할 단백질이 하나 또는 몇 개뿐일 때는 이 방법도 괜찮습니다.
  • 즐겨 사용하는 텍스트 편집기의 “찾기-바꾸기” 기능을 활용하기: 수동으로 세는 것보다 긴 단백질에 효과적이지만, 많은 단백질을 분석하려는 경우에는 역시 그리 편리하지 않습니다.
  • 컴퓨터 프로그램 작성하기: 이 장에서는 파이썬(Python) 언어로 작성된 프로그램을 사용할 것입니다. 박스 2.1에는 컴퓨터가 잔기(residue)를 어떻게 세는지에 대한 몇 가지 고찰이 담겨 있습니다.

다음 서열에 C가 몇 개나 있나요?

CCCHAJEAFIELAKJNFVLAIFEJLIEFJDCCCEFLEFJ

서열을 유심히 살펴보면 6개의 C가 있다는 것을 알 수 있습니다. 여러분은 직관적으로 어떻게 세어야 하는지 이해하고 정확한 결과를 얻었습니다. 하지만 컴퓨터에게 이 일을 대신 하도록 어떻게 말할 수 있을까요?

그 답에는 프로그래밍에 관한 많은 내용이 들어 있습니다. 먼저 해야 할 일을 완전히 이해한 다음, 그것을 정확하게 설명해야 합니다. 그렇다면 여러분은 정확히 어떻게 C를 셌나요? 아마도 다음 중 하나를 수행했을 것입니다.

  • 왼쪽에서 오른쪽으로 각 문자를 확인하며 나타나는 C를 모두 셌습니다.
  • 오른쪽에서 왼쪽으로 각 문자를 확인하며 나타나는 C를 모두 셌습니다.
  • 모든 문자를 세는 데 시간이 너무 오래 걸릴 것 같아 대략적인 추정치를 냈습니다.

직관이 없습니다. 스스로 어느 쪽에서 시작할지 판단할 수 없습니다. 여러분에게는 명백해 보이는 일이라도 컴퓨터는 여러분이 무엇을 기대하는지 스스로 결론 내릴 수 없습니다. 따라서 아주 사소한 부분까지 무엇을 해야 할지 말해주어야 합니다. 프로그램 코드(code)로 쉽게 번역될 수 있는 정확한 지침은 다음과 같습니다.

  2. 서열의 첫 번째 문자를 확인한다.
  3. 만약 그것이 C라면, 카운터에 1을 더한다.
  4. 마지막 문자에 도달했다면, 카운터 값을 출력하고 멈춘다.

프로그래밍의 많은 부분은 과업을 아주 작은 연산들로 쪼개는 것입니다. 세 번째 옵션은 예상치 못한 접근 방식인 추정을 사용합니다. 거대한 서열의 경우 참조 데이터를 통해 교육된 추측을 하는 것이 합리적일 수 있습니다. 단, 컴퓨터에게 어떻게 추측할지 알려주어야 하며 그 추측이 충분히 정확해야 한다는 조건이 붙습니다. 어떤 경우든 프로그래밍할 때는 직관에 반하는 해결책에 대해서도 열린 마음을 가지세요. 그것이 세는 것이든 추측하는 것이든, 일단 컴퓨터에게 무엇을 해야 할지 알려주면 컴퓨터는 그 일을 믿을 수 없을 정도로 빠르게 수행합니다. 여러분이 관심을 두는 대상이 하나의 짧은 서열이든, 수백 개의 서열이든, 혹은 전체 게놈(genome)이든 상관없습니다. 이 박스 처음에 나온 서열이 여러분이 직접 손으로 세어야 했던 마지막 서열이 되기를 바랍니다.

이전 장에서 여러분은 변수(Variable)에 데이터를 저장하고, 계산을 수행하고, 모듈(Module)을 가져와 사용하는 방법을 배웠습니다. 섹션 2.2.2의 파이썬 세션에서는 네 가지를 더 배우게 될 것입니다. 첫째, 파이썬 인터프리터(interpreter)가 실행하지 않도록 # 기호를 사용하여 코드 한 줄을 주석(comment) 처리하는 방법을 배웁니다. 둘째, 문자열(String)이라는 데이터 유형(data type)을 사용하여 변수에 텍스트를 저장하는 방법을 배웁니다. 아미노산을 세기 위해 단백질 문자열에 연결된 메서드(method, 데이터 객체에 연결된 함수)를 사용할 것입니다. 셋째, 동작을 여러 번 반복(repeat)하는 방법을 배웁니다. 이를 위해 for 루프(Loop)를 사용할 것입니다. 넷째이자 마지막으로, print 명령어를 사용하여 화면에 눈에 보이는 출력(Output)을 생성하는 방법을 살펴봅니다(박스 2.4 참조).
